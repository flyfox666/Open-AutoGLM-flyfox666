# 🚀 AutoGLM + scrcpy 联合启动

## ✨ 简单方案

不再需要复杂的集成，现在可以同时启动两个独立的程序：
- **AutoGLM Web UI**：智能语音/文本控制（已恢复原始版本）
- **scrcpy屏幕镜像**：实时屏幕显示和手动控制

## 🎯 启动方法

### 方法1：分别启动（推荐）
```bash
# 终端1：启动AutoGLM Web UI
python start_web_ui.py

# 终端2：启动scrcpy屏幕镜像
scrcpy-win64-v3.3.3\scrcpy-console.bat
```

### 方法2：使用批处理文件（Windows）
双击运行：`start_all.bat`

### 方法3：使用Python脚本
```bash
python start_all.py
```

## 📱 使用方式

启动后会出现两个窗口：

### 1. Web界面（AutoGLM）
- **访问地址**：http://localhost:8865
- **功能**：智能语音/文本控制
- **使用**：输入命令如"打开美团搜索火锅店"

### 2. scrcpy窗口（屏幕镜像）
- **功能**：实时显示设备屏幕
- **控制**：鼠标点击、键盘输入、手势操作
- **用途**：手动操作、查看实时屏幕

## 🔄 协同工作

两个程序可以同时使用：

- **智能控制**：通过Web界面发送语音命令
- **实时查看**：在scrcpy窗口查看命令执行结果
- **手动干预**：需要时可在scrcpy窗口手动操作

## ⚙️ 准备工作

### 1. 确保设备已连接
```bash
adb devices
```
应该显示您的设备

### 2. 确保依赖已安装
```bash
pip install -r requirements.txt
```

### 3. 确保scrcpy程序存在
检查路径：`scrcpy-win64-v3.3.3\scrcpy.exe`

## 🎮 优势

### 相比集成方案的优势：
1. **更稳定**：两个独立程序，互不干扰
2. **更简单**：无需复杂配置，即开即用
3. **更灵活**：可以单独使用任一功能
4. **更低延迟**：scrcpy原生运行，性能最佳

### 使用场景：
- **开发调试**：Web界面发送命令，scrcpy查看结果
- **演示展示**：Web界面智能控制，scrcpy实时展示
- **日常使用**：根据需要选择控制方式

## 🔧 故障排除

### Web界面无法访问
- 检查端口是否被占用
- 确认Gradio依赖已安装
- 查看控制台错误信息

### scrcpy无法启动
- 确认设备已连接并授权
- 检查USB调试是否开启
- 验证scrcpy程序路径

### 设备连接问题
1. 手机开启"开发者选项"
2. 开启"USB调试"
3. 连接时选择"允许USB调试"
4. 信任此电脑

## 💡 使用技巧

### Web界面技巧：
- 使用明确的命令描述
- 可以说"打开XX应用然后搜索YY"
- 支持多步骤操作

### scrcpy窗口技巧：
- 鼠标右键 = 返回键
- 鼠标中键 = 主页键
- Ctrl+C = 复制
- Ctrl+V = 粘贴
